## Anti-Patterns Verdict

**No AI-slop tells.** No gradient text, no glassmorphism beyond the legitimate sticky-header blur, no hero-metric stat tiles, no identical card grids. The eyebrow-label system and the 01–03/01–04 numbered lists are the closest things to "AI editorial scaffolding," but both map to genuinely enumerable content rather than decorative padding.

**Deterministic scan** (`detect.mjs`, exit 2, 3 findings): `single-font` (**false positive** — Spectral + Hanken Grotesk are both loaded via the Google Fonts `<link>` and wired through `--font-display`/`--font-sans`; confirmed live via computed styles), `em-dash-overuse` (6 counted, but most are in duplicated meta tags, not body copy — real usage is ~3-4 instances, consistent stylistic pattern, not AI cadence), `numbered-section-markers` (real — 01–03 and 01–04 lists both present, see P2 below).

**Browser evidence**: zero console errors, zero failed requests, all images/fonts load 200 OK. Accessibility tree: one `h1`, sensible heading nesting, all images have descriptive alt text, skip-link present and correctly targeted.

## Overall Impression

This is a well-crafted, restrained editorial page that doesn't read as AI-generated. The single biggest opportunity found during this session was the accent-color migration gap (now fixed live during this critique — see below) and tightening the hero's light-mode background contrast.

## What's Working

1. **Token discipline is real, not aspirational** — down to the wordmark's hand-tuned filter (`hue-rotate(287.5deg) saturate(0.68) brightness(0.43)`) shifting the mint PNG to the exact accent olive rather than shipping a second asset.
2. **Writing voice matches the stated brand personality** — "Nobody should need a manual for the tools they use every day" reads like an actual senior designer, not marketing copy.
3. **Dark-mode hero background execution is genuinely good** (fern motif fades to near-invisible against near-black) — proves the pattern works; made it easy to spot that light mode needed the same treatment (fixed below).

## Priority Issues

**[P0] Terracotta/olive dual-accent clash in the footer contact band — FIXED DURING THIS SESSION.**
Both design-review agents independently flagged this as the most severe issue on the page: `--band-bg` was still hardcoded terracotta (`#C56A3E`) while every other accent moment had migrated to olive, so the page's highest-stakes CTA (the footer "Say hello") showed a second, unrelated saturated hue. **Fix applied**: `--band-bg` is now `oklch(0.605 0.045 127.8)` / `#7A876A` (olive-family, ~4.57:1 against `--band-text`), verified live via computed styles and screenshot.

**[P1] Light-mode hero background reduced headline legibility.**
The fern motif's gradient overlay bottomed out at only 40% opacity in light mode vs. dark mode's more aggressive fade, letting linework cross directly through the headline letterforms. *Status: worth a follow-up pass* — the hero padding was tightened this session (see below) which changes the crop, but the fade-curve tuning itself hasn't been revisited; re-check after the padding change settles.

**[P2] Double numbered-list pattern (01–03 "What I Do" and 01–04 "How I Work") reads as repetition.**
Both sections use the identical serif-numeral-plus-title-plus-description device within three sections of each other. **Fix**: differentiate the second instance (drop numerals, use a different structural device) or merge/shorten one list.

**[P2] No active-section indicator in the sticky nav.**
A visitor who jumps via anchor link gets no "you are here" confirmation in the header nav. **Fix**: scroll-spy class toggle on `.site-header__link` for the section in view.

**[P3] Case-card eyebrow row wraps awkwardly at 375px** (`.case-card__eyebrow`, components.css:317) — client/theme/separator-dot can break mid-relationship on narrow viewports. **Fix**: stack client and theme on separate lines below 600px instead of relying on flex-wrap.

## Persona Red Flags

**Jordan (first-timer)**: The three "complexity" categories (Organizational/Behavioral/Interaction) are abstract nouns with no concrete example until the reader scrolls to the case cards — a skimmer who doesn't reach that point may read them as generic consultant-speak.

**Riley (stress-tester)**: The hero's glyph-scatter entrance animation (`initThesisSettle()`, js/app.js) self-modifies the DOM and only restores real text after the animation completes — rapid theme-toggling or navigating away mid-animation is an untested edge case worth a manual pass, especially for screen readers that read DOM mutations live rather than waiting for the `aria-label`.

**Sam (accessibility-dependent)**: Focus-visible override on the footer band correctly swaps to band-ink for contrast — a good, deliberate catch. The hero glyph-scatter animation is the one open question (see above).

## Minor Observations

- OG/Twitter image points to the square portrait rather than a purpose-built social card.
- `.about__credential` link uses plain browser underline rather than the more considered hover treatment used elsewhere (e.g. `.site-footer__link`).
- Copyright year is hardcoded "© 2026" — will need a manual bump or a `new Date().getFullYear()` swap.

## Questions to Consider

- If "one confident accent per theme" is the system's own stated rule, what allowed the footer band to ship terracotta through a full olive migration in the first place — is there a review step that should catch shared-token drift like this earlier?
- The hero's entrance animation delays legible text on *every* visit, not just first-time — for an audience explicitly described as scanning quickly, is it earning its keep?
