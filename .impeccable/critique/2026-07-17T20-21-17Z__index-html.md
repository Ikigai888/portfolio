## Anti-Patterns Verdict

**Start here.** Does this look AI-generated?

**LLM assessment**: Execution quality is genuinely high — real product screenshots, hand-tuned OKLCH contrast math, hardened breakpoints, deliberate off-reflex-list fonts (Spectral + Hanken Grotesk, neither on the ban list). But the aesthetic *lane* is exactly the one `reference/brand.md` names as currently saturated: "Editorial-typographic… display serif (often italic) + small tracked labels + ruled separators + monochromatic restraint… every Stripe-adjacent and Notion-adjacent brand has landed here." This site is: italic Spectral headline + uppercase 0.14em-tracked eyebrow labels + ruled separators + strict one-accent restraint. It clears the *first-order* reflex test (not purple-gradient SaaS-cream) but walks into the documented *second-order* trap. No side-stripe borders, no gradient text, no decorative glassmorphism, no hero-metric template, no identical icon-card grid — those all pass clean.

**Deterministic scan**: `node detect.mjs --json index.html` → exit code 2, 2 findings:
1. `single-font` (warning) — **false positive**, root-caused: the required parser deps (`htmlparser2`, `css-select`, `css-tree`, `domutils`) aren't installed in this environment, so the detector silently falls back to a regex-only engine that can't resolve `var(--font-display)` usage in the external stylesheets, and its Google-Fonts URL regex only captures the *first* `family=` param in a combined multi-family request — so it never sees Spectral. DESIGN.md's "Serif-For-Scale Rule" and real `font-family: var(--font-display)` usage on headlines/statements/case-card titles confirm two fonts are genuinely in play. This is an environment/tooling gap, not a real slop tell — worth flagging to whoever maintains the skill (missing npm deps, and a URL-parsing bug in the fallback engine's Google Fonts regex).
2. `numbered-section-markers` (advisory) — **likely false positive**. The only 01/02/03 markers on the page are item indices within two enumerated lists (3 "kinds of complexity," 3 case studies) — a legitimate 1:1 taxonomy mapping DESIGN.md documents by name ("Serif numerals" §3, "ledger rows" §5), not top-level *section* numbering used to fake structure. The detector's own advisory (not hard) severity reflects that this is a judgment call.

## Overall Impression

Craft is real: the contrast math checks out under independent recomputation, breakpoints are hardened against a documented 320px overflow bug, and touch targets are deliberately sized almost everywhere. But the page's actual weak point isn't sloppiness, it's **sameness** — six sections that all open with the identical tracked-eyebrow device read as templated regardless of how considered the underlying tokens are, and it's competing directly with the most saturated aesthetic lane in brand-register work right now. The single biggest opportunity: break the eyebrow-on-every-section rhythm so the page's genuine distinctiveness (real product screenshots, specific case-study questions, the drenched footer close) isn't undercut by a repeated structural tic.

## What's Working

1. **Contrast discipline is real, not just claimed.** Independently recomputed against the shipped hex/oklch values: text-primary/bg 14.55:1, text-muted/bg 5.27:1, accent-text/bg 6.79:1, band-text/band-bg 4.57:1 — every pairing DESIGN.md cites clears AA with margin.
2. **44px tap-target hardening on primary interactive elements**, explicitly commented in the CSS (nav links, hero CTA, footer links) — real accessibility discipline, not an afterthought.
3. **Case-study image alt text is genuinely descriptive**, not generic ("Agent-facing CMA in the Presentation Builder with the Style Kit switcher open, offering Primary, Luxury, and Commercial kits" — `index.html:116`), matching the brand register's "alt text is part of the voice" guidance.

## Priority Issues

**[P1] Theme toggle buttons are 24×24px, the smallest and most easily mis-tapped targets on the page, sitting 6px apart.**
- Why it matters: `css/components.css:169` sets `--toggle-size: 24px` with zero internal padding, landing exactly at the bare WCAG 2.5.8 AA floor — well under the ≥44px convention the rest of the site explicitly follows (hero CTA, nav links, footer links). Two adjacent 24px targets 6px apart is a real mobile mis-tap risk.
- Fix: grow the button to 32-36px with 4-6px padding, or expand the hit area asymmetrically (outer/top/bottom only) so the two targets stop crowding each other.
- Suggested command: `/impeccable audit` (or `/impeccable adapt` for the mobile-specific angle)

**[P1] Six of six homepage sections use the identical uppercase-tracked "eyebrow" label — the repetition-count AI-slop tell the parent skill names explicitly.**
- Why it matters: `SectionHeading()` fires on Hero, What I Do, Case Studies, How I Work, About, and Contact. The skill's own ban is explicit that documented intent doesn't exempt a pattern from the repetition-count failure mode. It also flattens the page's rhythm — every section opens the same way, undercutting the "authored, editorial" feel the brand is going for.
- Fix: vary the section-opening device — drop the eyebrow on 1-2 sections and let the statement/heading carry the opener alone, or swap one instance for a different structural cue (a pull quote, a different rule treatment).
- Suggested command: `/impeccable typeset` or `/impeccable layout`

**[P2] DESIGN.md's own "Hero (signature layout)" spec no longer matches what's shipped.**
- Why it matters: DESIGN.md describes a two-column hero (text left, portrait right) and a theme toggle with a visible "icon + eyebrow-style label." The shipped hero is single-column with no portrait (the portrait now lives in the About byline as a small avatar), and the toggle is icon-only. Orphaned `.hero__text` / `.hero__portrait` rules still sit in the 900px media query targeting classes absent from current markup — confirmed drift, not a one-off typo.
- Fix: update DESIGN.md §5 to match the shipped single-column hero and icon-only toggle; delete the orphaned media-query rules.
- Suggested command: `/impeccable document`

**[P2] No lower-commitment CTA for a hiring manager who isn't ready to email yet.**
- Why it matters: the only homepage conversions are "read a case study" or "email/call/LinkedIn." PRODUCT.md's stated success metric is "trusting his judgment, then clicking through" — a resume/PDF download is a natural, low-risk middle step that's currently absent for a visitor who wants to save or forward the profile.
- Fix: add a resume/CV link in the footer contact columns alongside Email/Phone/LinkedIn.
- Suggested command: `/impeccable clarify` or `/impeccable onboard`

**[P3] Footer CTA renders up to 112px (`7rem`), past the stated 96px (`6rem`) display-heading ceiling.**
- Why it matters: `.site-footer__cta` clamps to `clamp(2.75rem, 8vw, 7rem)`, 16px past the skill's general ceiling. This reads as a deliberate "drenched closer" and brand-register work is explicitly permitted bigger typographic swings than product UI — but it's worth a conscious call rather than a silent one.
- Fix: either lower the cap to 6rem, or note the ceiling exception explicitly in DESIGN.md's Do's/Don'ts so a future pass doesn't "fix" it by accident.
- Suggested command: `/impeccable typeset`

## Persona Red Flags

A first-time visitor has to guess sun=light/moon=dark from icon shape alone before ever clicking. Everything else (nav labels, CTA copy, case-study questions) is plain-language and clears Jordan's 5-second test easily.

**Riley (Deliberate Stress Tester)**: Genuinely hard to break — real hardening discipline shows up in the code (a documented, fixed 320px-viewport overflow bug behind `--fs-display`'s `min()` floor; global `overflow-wrap: break-word` as insurance against unbroken long strings). The one gap Riley would flag is the same one Casey hits: the theme-toggle's 24×24px targets are the most likely mis-tap/broken-interaction spot on the page under rapid or imprecise input.

**Casey (Distracted Mobile User)**: Touch targets are well-handled almost everywhere — 44px tap boxes are explicit and commented throughout the CSS — except the theme toggle, which is Casey's single worst spot on the page: two 24px icon buttons 6px apart is exactly the "tiny tap targets close together" red flag this persona test calls out by name. Primary actions (hero CTA, footer CTA) are large and thumb-friendly once scrolled to.
